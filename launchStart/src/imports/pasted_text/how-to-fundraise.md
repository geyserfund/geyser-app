Design a new responsive landing page for Geyser called:

“How to fundraise on Geyser”

A separate Geyser design system / implementation spec will be pasted below this prompt. Treat that spec as the visual and component source of truth for layout, typography, spacing, tokens, accessibility, and responsive behavior.

Important context:
- This page is for creators.
- It should feel friendly, inspiring, simple, and encouraging.
- It should not feel technical, dense, or overwhelming.
- It should not feel like documentation.
- It should not repeat the emotional/community page about “Join the Geyser creator community.”
- This page is a sales page first, with light guidance built in.
- The main goal is to make people want to click “Start your project.”

Core idea:
This page should help people feel:
- I can do this
- Geyser makes this feel approachable
- I understand the basics
- I know where to learn more
- I’m ready to start

Tone:
- warm
- clear
- creator-friendly
- motivating
- trustworthy
- simple
- action-oriented

Do not over-explain.
Do not add too much product detail.
Do not make the top half technical.
Keep the copy light and emotionally confident.
Use short sections that each explain one important part of running a fundraiser.
Each section should include a small, visually integrated link to the relevant guide for people who want the deeper explanation.

Primary CTA throughout:
- Start your project

Secondary CTA options:
- Read the guide
- Learn more
- See launch plans

Overall page direction:
This should feel like a polished launch invitation with practical confidence-building.
It should read like:
“Here’s how great fundraising works on Geyser — and you can start today.”

- clean hierarchy
- soft but confident visuals
- simple product mockups
- creator-friendly illustrations or images
- short sections
- brief guidance
- subtle trust signals
- repeated CTA rhythm
- light guide-link moments in each section

Do not build this like a giant step-by-step manual.
Think:
